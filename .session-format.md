# ssqq 세션 로그 포맷

앞으로 `ssqq/work_log.md` 에 추가되는 모든 entry는 이 템플릿을 따른다.
사용자 요청 (2026-04-08): "시작 전과 끝나고 문제점들 계속 작업을 적으면서 진행".

---

## 템플릿

```
### YYYY-MM-DD (선택: 오전/오후/세션 slug)

**범위**: [git-only | ops-drift | mixed]
**브랜치**: work/... (또는 main 금지)
**drift 발생**: yes/no  → yes면 ops_drift_log.md 참조 줄 추가

**시작 전 상태**:
- git: <브랜치, 최신 커밋 hash, ahead/behind>
- ops: <운영 관련 기준선 1~3줄>
- 관련 파일 상태: <대상 파일의 현재 상태>

**작업 내용**:
1. <순서대로 번호 bullet>
2. ...

**발견된 문제/이슈**:
- [해결] <내용 + fix 위치>
- [잔존] <내용 + 영향/다음 단계>
- 없으면 "없음" 한 줄

**끝난 후 상태**:
- git: <새 커밋 hash, ahead 수>
- ops: <변경이 있다면, 0이면 "0 변경">
- 관련 파일 상태: <대상 파일의 새 상태>

**변경 파일** (git 기준):
- ssqq/경로 (신규/수정/삭제)

**검증**:
- <실제 실행한 검증 명령 + 결과 요약, 1~5줄>

**커밋**:
- <hash> <subject>
- (여러 개면 나열)
```

## 필드별 규칙

### `범위` 태그
- **git-only**: 로컬 파일/git commit만. 운영 컨테이너/DB 0 변경. 가장 안전.
- **ops-drift**: 운영 쪽 직접 변경 포함 (컨테이너 패치/배포/DB 수정 등). 반드시 `ops_drift_log.md` 동반 entry.
- **mixed**: git + ops 동시. 둘 다 기록.

### `drift 발생` = yes 인 경우
반드시 다음 중 최소 하나 수행:
1. `ssqq/ops_drift_log.md` 에 entry 추가 (같은 커밋 또는 별도 커밋)
2. 또는 entry 본문에 "drift 상세: ops_drift_log.md#<anchor>" 링크

### `발견된 문제/이슈` 태그
- `[해결]`: 세션 내에서 처리 완료. fix 위치(파일:라인) 명시.
- `[잔존]`: 다음 세션/PM으로 넘어감. 영향 범위와 다음 단계 기록.
- `[관찰]`: 당장 작업은 아니지만 기록해둘 이상 동작.

### `검증` 섹션
"실제로 어떻게 확인했는지" 구체적으로:
- 실행한 명령 (`git log`, `curl`, `docker exec ... psql`, `diff`)
- 결과 요약 (OK / 실패 / 수치)
- 경계 0 변경 검증 결과 (해당되는 경우)

### `커밋` 섹션
- 커밋 후 `git log --oneline -1` 로 hash 확인해서 채움
- 커밋 안 했으면 "없음 (로컬 파일 작업만)"

## 세션 시작 체크리스트

1. `ssqq/work_log.md` 최근 entry 읽기 (어제 뭐 했는지)
2. `ssqq/ops_drift_log.md` 읽기 (운영 drift 상태)
3. `git status` / `git log --oneline -5` (현재 브랜치 + ahead/behind)
4. 작업 범위가 ssqq/ 경계 내인지 확인
5. 필요하면 `.session-format.md` 템플릿을 메모리에 로드

## 세션 종료 체크리스트

1. `work_log.md`에 오늘 entry 추가했나? (위 템플릿으로)
2. drift 발생했으면 `ops_drift_log.md`에도 entry 추가했나?
3. `git status` clean? (untracked 남았으면 의도한 것인지 명시)
4. 커밋 완료? 커밋 메시지 Conventional Commits (`feat:/fix:/docs:/refactor:/build:`)?
5. `git push` 필요? (필요하면 사용자에게 확인 또는 허락된 경우 push)
6. 다음 세션용 "현재 상태" 한 줄이 entry 끝에 기록됐나?

## 경계 규칙 (절대)

- `ssqq/` 폴더 밖 변경 금지 (DeliveryOrderReceiver/ v2, server/, claude-1 루트)
- `main` 브랜치 직접 커밋 금지, 항상 work 브랜치
- `git push --force` 금지 (fast-forward만)
- 운영 컨테이너 직접 패치 시 반드시 `ops_drift_log.md` 기록 + PM 허락 전제
- 기존 drift 원복/재패치 금지 (기록만)

## 관련 파일

- `ssqq/work_log.md` — 이 템플릿을 사용하는 실제 로그
- `ssqq/ops_drift_log.md` — 운영 drift 단일 기록소
- `ssqq/dev_progress.md` — 기획서 / 프로젝트 상태 (수동 업데이트)
- `ssqq/issue_matrix.md` — 이슈 매트릭스 (수동 업데이트)
